## Docker 설치

### Windows

1. **WSL2 설치** (Windows 10/11)
   ```powershell
   wsl --install
   ```
   재부팅 후:
   ```powershell
   wsl --set-default-version 2
   ```

2. **Docker Desktop 설치**
   - [Docker Desktop for Windows](https://www.docker.com/products/docker-desktop/) 다운로드
   - 설치 시 "Use WSL 2 instead of Hyper-V" 옵션 선택
   - 설치 완료 후 재부팅

3. **Docker Desktop 설정**
   - Docker Desktop 실행
   - **Settings** → **General**
     - ✅ Use the WSL 2 based engine
   - **Settings** → **Resources** → **WSL Integration**
     - ✅ Enable integration with my default WSL distro
   - **Settings** → **Resources** → **File Sharing**
     - 프로젝트 디렉토리 경로 추가 (예: `C:\workspace`)
   - **Apply & Restart**

4. **설치 확인**
   ```powershell
   docker --version
   # 출력 예: Docker version 24.0.7, build afdd53b

   docker run hello-world
   # "Hello from Docker!" 메시지 확인
   ```

### Mac

1. **Docker Desktop 설치**
   - [Docker Desktop for Mac](https://www.docker.com/products/docker-desktop/) 다운로드
   - Intel Chip: `Docker Desktop for Mac (Intel chip)`
   - Apple Silicon: `Docker Desktop for Mac (Apple chip)`
   - 설치 후 Docker Desktop 실행

2. **설치 확인**
   ```bash
   docker --version
   docker run hello-world
   ```

### Linux (Ubuntu/Debian)

1. **Docker Engine 설치**
   ```bash
   # 기존 Docker 제거
   sudo apt-get remove docker docker-engine docker.io containerd runc

   # 의존성 설치
   sudo apt-get update
   sudo apt-get install -y \
       ca-certificates \
       curl \
       gnupg \
       lsb-release

   # Docker GPG 키 추가
   sudo mkdir -p /etc/apt/keyrings
   curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \
       sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

   # Docker repository 추가
   echo \
     "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
     https://download.docker.com/linux/ubuntu \
     $(lsb_release -cs) stable" | \
     sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

   # Docker 설치
   sudo apt-get update
   sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

   # 현재 사용자를 docker 그룹에 추가
   sudo usermod -aG docker $USER
   newgrp docker
   ```

2. **설치 확인**
   ```bash
   docker --version
   docker run hello-world
   ```

## 개발 컨테이너 설정

### 방법 1: Dockerfile로 이미지 빌드 (권장)

#### 1단계: 이미지 빌드

프로젝트 루트에서 실행:

**Windows (PowerShell):**
```powershell
cd C:\workspace\monorepo
docker build -t monorepo-dev -f .devcontainer/Dockerfile .
```

**Mac/Linux:**
```bash
cd ~/workspace/monorepo
docker build -t monorepo-dev -f .devcontainer/Dockerfile .
```

빌드 시간: 약 5-10분 (네트워크 속도에 따라 다름)

#### 2단계: 컨테이너 실행

**Windows (PowerShell):**
```powershell
# 기존 컨테이너 제거 (있을 경우)
docker rm -f monorepo-dev 2>$null

# 컨테이너 실행
docker run -it -d `
  -v "${PWD}:/workspace" `
  -w /workspace `
  -p 3000:3000 `
  -p 3001:3001 `
  -p 8080:8080 `
  -p 8081:8081 `
  -p 8082:8082 `
  -p 8083:8083 `
  --name monorepo-dev `
  monorepo-dev
```

**Mac/Linux:**
```bash
# 기존 컨테이너 제거 (있을 경우)
docker rm -f monorepo-dev 2>/dev/null

# 컨테이너 실행
docker run -it -d \
  -v "$(pwd):/workspace" \
  -w /workspace \
  -p 3000:3000 \
  -p 3001:3001 \
  -p 8080:8080 \
  -p 8081:8081 \
  -p 8082:8082 \
  -p 8083:8083 \
  --name monorepo-dev \
  monorepo-dev
```

#### 3단계: VSCode에서 컨테이너 연결

1. VSCode 실행
2. `Ctrl+Shift+P` (Mac: `Cmd+Shift+P`)
3. **"Dev Containers: Attach to Running Container"** 선택
4. **monorepo-dev** 컨테이너 선택
5. 새 VSCode 창에서 **File** → **Open Folder** → `/workspace` 입력

### 방법 2: Dev Container 자동 설정

VSCode Dev Containers 확장이 `.devcontainer/devcontainer.json`을 자동으로 감지합니다:

1. VSCode로 프로젝트 폴더 열기
2. 좌측 하단 파란색 버튼 클릭 또는 `Ctrl+Shift+P`
3. **"Dev Containers: Reopen in Container"** 선택
4. 자동으로 이미지 빌드 및 컨테이너 실행

## 의존성 설치

VSCode 터미널에서 실행 (컨테이너 내부):

### 1. 환경 확인

```bash
# 위치 확인
pwd
# 출력: /workspace

# 도구 버전 확인
go version
# 출력: go version go1.23.8 linux/amd64

python3 --version
# 출력: Python 3.11.15

node --version
# 출력: v18.20.8

pnpm --version
# 출력: 9.15.9
```

### 2. Pants 빌드 시스템 설치

이 프로젝트는 **scie-pants 런처**를 사용합니다. Pants 2.18 이후 버전은 GitHub Releases에서 배포되며, scie-pants 런처가 자동으로 다운로드합니다.

**특징:**
- ✅ Python 인터프리터 내장 - Python 버전 불일치 문제 없음
- ✅ GitHub에서 자동 다운로드 - PyPI 제한 없음
- ✅ 프로젝트별 독립 실행 - 여러 Pants 버전 동시 사용 가능
- ✅ 자동 의존성 격리 - `.venv` 가상환경 불필요

**가상환경이 필요 없는 이유:**

Pants는 내부적으로 모든 Python 의존성을 **자동으로 격리된 환경**에서 관리합니다:
- 각 타겟마다 독립된 PEX 환경 생성
- 의존성 충돌 자동 해결
- 캐시된 환경 재사용으로 빠른 실행
- 위치: `/root/.cache/pants/` 및 `/root/.cache/nce/`

따라서 `python -m venv .venv` 또는 `virtualenv` 설정이 **필요하지 않습니다**.

**설치 확인:**
```bash
# 프로젝트에 이미 포함되어 있음
ls -la pants
# -rwxr-xr-x 1 root root 7.2M ... pants

# 버전 확인
./pants --version
# 2.30.1
```

**참고:** 컨테이너 이미지에는 이미 scie-pants가 포함되어 있습니다. 로컬에서 사용하려면:
```bash
# Linux (x86_64)
curl -L https://github.com/pantsbuild/scie-pants/releases/download/v0.12.5/scie-pants-linux-x86_64 -o pants
chmod +x pants

# macOS (Apple Silicon)
curl -L https://github.com/pantsbuild/scie-pants/releases/download/v0.12.5/scie-pants-macos-aarch64 -o pants
chmod +x pants

# macOS (Intel)
curl -L https://github.com/pantsbuild/scie-pants/releases/download/v0.12.5/scie-pants-macos-x86_64 -o pants
chmod +x pants
```

### 3. Makefile로 전체 의존성 설치

```bash
make install
```

이 명령은 다음을 순차적으로 실행합니다:
1. **Pants 버전 확인**: pants.toml 설정대로 자동 설치
2. **Python 의존성**: lockfile 생성
3. **Web 의존성**: pnpm install via Pants

### 4. 개별 설치 (선택사항)

필요시 개별적으로 설치:

```bash
# Pants만 설치
make install-pants

# Python 의존성만 생성
make install-python-deps

# Web 의존성만 설치
make install-web
```

### 5. 설치 확인

```bash
# Pants 확인
./pants --version
# 출력: 2.30.1

# Python 타겟 확인
./pants list services/python::
# 출력: services/python/triage:triage, services/python/copilot:copilot, ...

# Web 타겟 확인
./pants list :: | grep web
# 출력: //:web-build, //:web-lint, //:web-test, ...

# Node modules 확인
ls node_modules/.bin/turbo
# 출력: node_modules/.bin/turbo
```

## 빌드 시스템 사용법

### 아키텍처 개요

```
┌─────────────────────────────────────────────┐
│              Makefile (Root)                │
│  - 전체 프로젝트 통합 빌드 인터페이스         │
└─────────────┬───────────────────────────────┘
              │
       ┌──────┴──────┐
       │             │
       ▼             ▼
┌────────────┐  ┌──────────────────────┐
│ Go Build   │  │   Pants Build        │
│ (Makefile) │  │   (Python + Web)     │
└────────────┘  └──────────────────────┘
       │                   │
       ▼                   ▼
  ┌─────────┐    ┌─────────┬──────────┐
  │ gateway │    │ Python  │   Web    │
  │detection│    │services │  (pnpm + │
  └─────────┘    └─────────┤  turbo)  │
                            └──────────┘
```

### Makefile 명령어 (통합 인터페이스)

#### 전체 명령어 보기

```bash
make help
```

#### 빌드

```bash
# 전체 빌드 (Go + Python + Web)
make build

# Go 서비스만 빌드
make go-build
# 결과: dist/go/gateway, dist/go/detection

# Python 서비스만 빌드
make python-build
# 결과: dist/*.pex 파일

# Web만 빌드
make web-build
# 결과: web/apps/*/.next/
```

#### 테스트

```bash
# 전체 테스트
make test

# 개별 테스트
make go-test
make python-test
make web-test
```

#### 린트 & 타입 체크

```bash
# 전체 린트
make lint

# 개별 린트
make go-lint           # go fmt + go vet
make python-lint       # ruff
make web-lint          # eslint

# 타입 체크
make python-check      # mypy
make web-check-types   # tsc
```

### Pants 직접 사용 (Python + Web)

Makefile 대신 Pants를 직접 사용할 수도 있습니다:

#### Python 서비스

```bash
# 모든 Python 타겟 보기
./pants list services/python::

# 특정 서비스 빌드 (PEX 파일 생성)
./pants package services/python/triage:triage
./pants package services/python/copilot:copilot
./pants package services/python/agents/investigation:investigation

# 결과 확인
ls dist/
# triage.pex, copilot.pex, investigation.pex

# Python 테스트
./pants test services/python::
./pants test services/python/triage::

# Python 린트
./pants lint services/python::

# Python 타입 체크
./pants check services/python::
```

#### Web Frontend

Web은 Pants의 `run_shell_command` 타겟으로 래핑되어 있습니다:

```bash
# Web 타겟 보기
./pants list :: | grep web
# //:web-install
# //:web-build
# //:web-lint
# //:web-test
# //:web-check-types
# //:web-dev

# Web 의존성 설치
./pants run //:web-install

# Web 빌드 (Turborepo 사용)
./pants run //:web-build

# Web 린트
./pants run //:web-lint

# Web 타입 체크
./pants run //:web-check-types

# Web 테스트
./pants run //:web-test
```

### Go 서비스 직접 빌드

Go 서비스는 Pants와 독립적으로 Go 툴체인을 직접 사용합니다:

```bash
# Gateway 서비스 빌드
cd services/go/gateway
go build -o ../../../dist/go/gateway ./cmd/gateway

# Detection 서비스 빌드
cd services/go/detection
go build -o ../../../dist/go/detection ./cmd/detection

# 또는 루트에서
make go-build

# 결과 확인
ls dist/go/
# gateway, detection
```

## 문제 해결

### Docker 관련

#### Docker Desktop이 시작되지 않음 (Windows)

**증상:**
```
Docker Desktop starting...
```
에서 멈춤

**해결:**
```powershell
# WSL 재시작
wsl --shutdown
wsl

# Docker Desktop 재시작
```

#### 컨테이너가 바로 종료됨

**증상:**
```bash
docker ps
# monorepo-dev가 목록에 없음
```

**해결:**
```bash
# 로그 확인
docker logs monorepo-dev

# 컨테이너 재시작
docker start monorepo-dev

# 또는 새로 생성
docker rm monorepo-dev
# 위의 docker run 명령어 다시 실행
```

#### 볼륨 마운트 권한 오류 (Linux)

**증상:**
```
Permission denied
```

**해결:**
```bash
# 프로젝트 디렉토리 권한 확인
ls -la

# 소유권 변경 (필요시)
sudo chown -R $USER:$USER .

# 컨테이너 재시작
docker restart monorepo-dev
```

### Pants 관련

#### Pants 버전 오류

**증상:**
```
Please explicitly specify the `pants_version` in your `pants.toml`
```

**해결:**
```bash
# pants.toml 확인
cat pants.toml | grep pants_version
# 출력: pants_version = "2.30.1"

# scie-pants 런처 재다운로드 (GitHub에서)
curl -L https://github.com/pantsbuild/scie-pants/releases/download/v0.12.5/scie-pants-linux-x86_64 -o pants
chmod +x pants
./pants --version
```

#### Python 의존성 오류

**증상:**
```
ModuleNotFoundError: No module named 'fastapi'
```

**해결:**
```bash
# requirements.txt 확인
cat requirements.txt

# lockfile 재생성
./pants generate-lockfiles

# 또는
make install-python-deps
```

#### Pants가 web 타겟을 찾지 못함

**증상:**
```
The address //:web-build does not exist
```

**해결:**
```bash
# BUILD.web 파일 확인
ls -la BUILD.web

# 타겟 목록 확인
./pants list ::

# pants.toml에 web이 source root에 포함되어 있는지 확인
cat pants.toml | grep -A5 "\[source\]"
```

### pnpm/Web 관련

**증상:**

**해결:**

```bash
pnpm install
```

**증상:**

**해결:**
```bash
# 타입 정의 설치

# 개별 앱 타입 체크

```

### Go 관련

#### Go 버전 오류

**증상:**
```
go: go.mod requires go >= 1.21
```

**해결:**
```bash
# Go 버전 확인
go version
# go version go1.23.8 linux/amd64

# Dockerfile에서 지정된 Go 사용
/usr/local/go/bin/go version
```

#### Go 모듈 의존성 오류

**증상:**
```
no required module provides package github.com/...
```

**해결:**
```bash
cd services/go/gateway
go mod tidy
go mod download

cd ../detection
```


#### Git 파일 권한 문제

**증상:**
```
warning: could not open directory: Permission denied
```

**해결:**
```bash
# Git 설정
git config --global --add safe.directory /workspace

# 또는
sudo chown -R $USER:$USER /workspace
```

## 컨테이너 관리 명령어

### 기본 명령어

```bash
# 실행 중인 컨테이너 확인
docker ps

# 모든 컨테이너 확인 (중지된 것 포함)
docker ps -a

# 컨테이너 시작
docker start monorepo-dev

# 컨테이너 중지
docker stop monorepo-dev

# 컨테이너 재시작
docker restart monorepo-dev

# 컨테이너 삭제
docker rm monorepo-dev

# 이미지 삭제
docker rmi monorepo-dev
```

### 컨테이너 접속

```bash
# Bash 쉘로 접속
docker exec -it monorepo-dev bash

# 특정 명령어 실행
docker exec monorepo-dev pwd
docker exec monorepo-dev ls -la

# 로그 확인
docker logs monorepo-dev
docker logs -f monorepo-dev  # follow mode
```

### 이미지 관리

```bash
# 로컬 이미지 목록
docker images

# 이미지 빌드
docker build -t monorepo-dev:latest -f .devcontainer/Dockerfile .

# 이미지에 태그 추가
docker tag monorepo-dev:latest monorepo-dev:v1.0

# 현재 컨테이너 상태를 이미지로 저장
docker commit monorepo-dev monorepo-dev:snapshot

# 이미지 내보내기
docker save monorepo-dev:latest -o monorepo-dev.tar

# 이미지 가져오기
docker load -i monorepo-dev.tar
```

### 리소스 정리

```bash
# 중지된 모든 컨테이너 삭제
docker container prune

# 사용하지 않는 이미지 삭제
docker image prune

# 사용하지 않는 볼륨 삭제
docker volume prune

# 모든 리소스 정리 (주의!)
docker system prune -a
```

## CI/CD 환경

### GitHub Actions 예시

```yaml
name: CI

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v3

    - name: Set up Docker
      uses: docker/setup-buildx-action@v2

    - name: Build dev image
      run: docker build -t monorepo-dev -f .devcontainer/Dockerfile .

    - name: Run CI checks
      run: |
        docker run --rm -v $(pwd):/workspace -w /workspace monorepo-dev \
          bash -c "make ci-all"
```

```bash
# 전체 CI 체크

make ci-web      # Web: lint + type-check + build + test
```

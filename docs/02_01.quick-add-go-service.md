# Go 서비스 추가 - 빠른 참조

새로운 Go 서비스를 `services/go`에 추가하는 빠른 가이드입니다.

---

## 빠른 시작 (5분)

서비스 이름: `analytics` (예시)

```bash
cd /workspace/services/go

# 1. 디렉토리 생성
mkdir -p analytics/cmd/analytics analytics/internal/handler

# 2. go.mod 생성
cd analytics
cat > go.mod <<'EOF'
module github.com/example/monorepo/services/go/analytics

go 1.23

require github.com/example/monorepo/services/go/pkg v0.0.0
EOF

# 3. go.work에 추가
cd /workspace/services/go
go work use ./analytics

# 4. main.go 작성
cat > analytics/cmd/analytics/main.go <<'EOF'
package main

import (
	"fmt"
	"log"
	"net/http"
)

func main() {
	http.HandleFunc("/health", func(w http.ResponseWriter, r *http.Request) {
		fmt.Fprintf(w, "OK")
	})

	log.Println("Starting analytics service on :8084")
	log.Fatal(http.ListenAndServe(":8084", nil))
}
EOF

# 5. BUILD 파일 생성 (문서화용)
cat > analytics/BUILD <<'EOF'
# Analytics service - built via Makefile
EOF

# 6. 의존성 정리
cd analytics
go mod tidy

cd /workspace/services/go
go work sync

# 7. 빌드 테스트
cd analytics
go build -o ../../dist/go/analytics ./cmd/analytics

# 8. 실행 테스트
../../dist/go/analytics &
curl http://localhost:8084/health
# 출력: OK

# 프로세스 종료
pkill analytics
```

## Makefile 통합 (필수)

`/workspace/Makefile` 수정:

### go-build 타겟

```makefile
go-build: ## Build Go services
	@echo "$(CYAN)Building Go services...$(RESET)"
	@mkdir -p dist/go
	@cd services/go/gateway && $(GO) build -o ../../../dist/go/gateway ./cmd/gateway
	@cd services/go/detection && $(GO) build -o ../../../dist/go/detection ./cmd/detection
	@cd services/go/analytics && $(GO) build -o ../../../dist/go/analytics ./cmd/analytics  # 추가
	@echo "$(GREEN)✓ Go services built$(RESET)"
```

### go-test 타겟

```makefile
go-test: ## Run Go tests
	@echo "$(CYAN)Running Go tests...$(RESET)"
	@cd services/go && $(GO) test ./gateway/... ./detection/... ./analytics/... ./pkg/...  # 추가
```

### go-lint 타겟

```makefile
go-lint: ## Lint Go code
	@echo "$(CYAN)Linting Go code...$(RESET)"
	@cd services/go && $(GO) fmt ./gateway/... ./detection/... ./analytics/... ./pkg/...  # 추가
	@cd services/go && $(GO) vet ./gateway/... ./detection/... ./analytics/... ./pkg/...  # 추가
```

### 개발 서버 타겟 (선택)

```makefile
go-dev-analytics: ## Run analytics service in dev mode
	@echo "$(CYAN)Starting analytics service on :8084...$(RESET)"
	@cd services/go/analytics && $(GO) run ./cmd/analytics
```

## 권장 디렉토리 구조

```
analytics/
├── BUILD                         # 문서화용 (Pants 사용 안 함)
├── go.mod                        # Go 모듈 정의
├── cmd/
│   └── analytics/
│       └── main.go              # 엔트리포인트
├── internal/                     # 외부 import 불가
│   ├── handler/
│   │   └── handler.go
│   ├── service/
│   │   └── service.go
│   └── storage/
│       └── storage.go
└── {pkg}/                        # 외부 import 가능 (선택)
    └── client/
        └── client.go
```

## 공유 패키지 사용

### 1. go.mod에 의존성 추가

```go
module github.com/example/monorepo/services/go/analytics

go 1.23

require github.com/example/monorepo/services/go/pkg v0.0.0
```

### 2. 코드에서 import

```go
import (
	"github.com/example/monorepo/services/go/pkg/common"
	"github.com/example/monorepo/services/go/pkg/utils"
)
```

### 3. workspace sync

```bash
cd /workspace/services/go
go work sync
```

## 문제 해결 (1분)

### 모듈을 찾을 수 없음

```bash
cd /workspace/services/go
go work sync
cd analytics
go mod tidy
```

### 빌드 오류

```bash
# 전체 workspace 빌드 테스트
cd /workspace/services/go
go build ./...
```

### Makefile 빌드 실패

```bash
# Go 경로 확인
make -n go-build

# 직접 빌드 테스트
/usr/local/go/bin/go version
cd services/go/analytics
/usr/local/go/bin/go build ./cmd/analytics
```

## 추가 설정 (선택)

### Docker 포트 추가

컨테이너 실행 시:

```bash
docker run -it -d \
  -v "$(pwd):/workspace" \
  -p 8084:8084 \  # 추가
  --name monorepo-dev \
  monorepo-dev
```

### .gitignore 확인

이미 설정되어 있어야 함:

```gitignore
# Go binaries
dist/go/*

# Go build cache
services/go/*/vendor/
```

## 명령어 요약

```bash
# 새 서비스 생성
cd /workspace/services/go
mkdir -p {service}/cmd/{service} {service}/internal

# Go 모듈 초기화
cd {service}
go mod init github.com/example/monorepo/services/go/{service}

# workspace 추가
cd /workspace/services/go
go work use ./{service}

# 의존성 정리
cd {service}
go mod tidy
cd ..
go work sync

# 빌드
cd {service}
go build -o ../../dist/go/{service} ./cmd/{service}

# 테스트
go test ./...

# Makefile 통합
# → go-build, go-test, go-lint 타겟에 추가
```
